# Phase 1 — UI Design Contract

> Visual and interaction contract for this phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** Phase 1 delivers no UI rendering — it is a pure infrastructure and backend proxy layer. This contract establishes the design system baseline that Phase 2 and Phase 3 consume. The executor running plan 01-01 will scaffold the project and run `npx shadcn@latest init`, at which point `components.json` will exist and `shadcn_initialized` becomes `true` for downstream phases.

### shadcn Initialization Parameters (for plan 01-01 executor)

Run after `create-next-app`:

```bash
npx shadcn@latest init -t next
```

When prompted, select:
- **Style:** New York
- **Base color:** Neutral
- **CSS variables:** Yes

Accept the default alias detection — shadcn reads `tsconfig.json` and pre-fills `@/*` → `./src/*` from the `create-next-app` flag `--import-alias "@/*"`. Do not override.

After init, confirm `src/components/ui/` exists and `components.json` is present at project root.

### Components Needed in Phase 1

None. Phase 1 installs the system only. No `npx shadcn add` calls until Phase 2.

## Spacing Scale

Declared values (multiples of 4 only). Source: 8-point grid default — no upstream artifact overrides.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (record row gaps inside cards) |
| md | 16px | Default element spacing (card internal padding, form field gaps) |
| lg | 24px | Section padding (card header-to-body, grid gap between cards) |
| xl | 32px | Layout gaps (page horizontal padding on desktop) |
| 2xl | 48px | Major section breaks (search bar margin-bottom before card grid) |
| 3xl | 64px | Page-level spacing (top-of-page vertical padding) |

Exceptions: Touch targets for icon-only buttons (copy-to-clipboard, Phase 3) — minimum 44×44px regardless of visual size. Implemented via padding, not by breaking the spacing scale.

## Typography

shadcn New York + Geist Sans. Exactly 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Card record values, metadata, error messages |
| Label | 12px | 400 (regular) | 1.4 | Record type badges (A, AAAA, MX), secondary labels |
| Heading | 16px | 600 (semibold) | 1.3 | Card titles (Geolocation, DNS Records, Reverse DNS, WHOIS) |
| Display | 24px | 600 (semibold) | 1.2 | Page title / app name |

Font stack: `var(--font-geist-sans), ui-sans-serif, system-ui, sans-serif`

Monospace exception: DNS record values (hostnames, IP addresses, PTR strings, nameservers) use `font-mono` at 13px / weight 400 / line-height 1.5 for scanability. This is a rendering decision for Phase 2 card components, not a new type scale size.

## Color

shadcn New York with Neutral base. CSS variables — both light and dark mode values are set by shadcn init. Source: CONTEXT.md (CSS variables enabled), RESEARCH.md (shadcn New York, Neutral base).

| Role | CSS Variable | Light Mode Hex | Dark Mode Hex | Usage |
|------|-------------|----------------|---------------|-------|
| Dominant (60%) | `--background` | #FAFAFA (neutral-50) | #09090B (neutral-950) | Page background, main surface |
| Secondary (30%) | `--card` | #FFFFFF (white) | #18181B (neutral-900) | Card backgrounds (all 4 lookup cards) |
| Accent (10%) | `--primary` | #18181B (neutral-900) | #FAFAFA (neutral-50) | Search button, focused input ring |
| Muted | `--muted` | #F4F4F5 (neutral-100) | #27272A (neutral-800) | Skeleton loading backgrounds |
| Destructive | `--destructive` | #EF4444 (red-500) | #EF4444 (red-500) | Error state card borders and icons only |

Accent reserved for: search button fill, focused input ring, active state indicators. Never used for card decorations, text, or hover states on non-interactive elements.

Destructive reserved for: inline error state display within cards (error icon + border tint). No destructive actions in Phase 1 (no user-facing destructive interactions until any future phase).

## Copywriting Contract

Phase 1 has no UI to render. This contract defines the copy for Phase 2 card and search bar components so the executor has it from the start. Source: REQUIREMENTS.md descriptions, CONTEXT.md error shape (`ApiError.message` field content).

| Element | Copy |
|---------|------|
| Primary CTA | **Search** (search bar submit button — single word, imperative verb) |
| Empty state heading | No results yet |
| Empty state body | Enter an IP address or domain above to run all lookups. |
| Private IP error | This is a private or reserved IP address — no external data is available. |
| Upstream API error | {Card name} lookup failed. Try again or check your input. |
| RDAP not supported | WHOIS is not available for this domain's TLD via RDAP. |
| Invalid input | Enter a valid IP address or domain name. |
| Input placeholder | IP address or domain (e.g. 8.8.8.8 or example.com) |
| Card: Reverse DNS (hidden for domain input) | — (card is hidden, no copy needed) |
| Destructive confirmation | None — no destructive actions in Phase 1 |

Error copy strategy: all `ApiError.message` strings are written above and go into `src/lib/api-error.ts` constants or inline in route handlers. Phase 2 renders them verbatim from the API response — no second copy layer.

## Interaction States (for Phase 2 executor reference)

These are declared here so Phase 2 has the full contract without needing to revisit this document.

| Component | States |
|-----------|--------|
| Search input | default, focused (primary ring), disabled (during in-flight lookup) |
| Search button | default, hover (muted shift), loading (spinner, disabled), active |
| Lookup card | idle/empty, loading (skeleton), error (destructive border tint), data |
| Copy button (Phase 3) | default, hover, copied (brief checkmark, 1.5s) |

Skeleton: use `--muted` background with CSS `animate-pulse`. Do not use third-party skeleton libraries.
