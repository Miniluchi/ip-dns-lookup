## Spacing Scale

Declared values (multiples of 4 — Tailwind default scale, no custom config file; tokens expressed as Tailwind utility class equivalents):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps (CopyButton gap, icon row gap) |
| sm | 8px | `gap-2`, `p-2` | Inline button padding, compact element spacing |
| md | 16px | `gap-4`, `p-4` | Default card padding, form element spacing |
| lg | 24px | `gap-6`, `p-6` | Card section padding (existing shadcn Card default) |
| xl | 32px | `gap-8`, `p-8` | Card grid gap |
| 2xl | 48px | `py-12` | Page vertical padding |
| 3xl | 64px | `py-16` | Not used in this phase |

Exceptions:
- Copy button touch target: `size-6` (24px) — ghost button with `size="icon"` prop; smaller than 44px acceptable because this is a pointer-device personal tool (RESEARCH.md Pitfall 4 — mobile hover accepted as known gap)
- Theme toggle button: `size-9` (36px) — shadcn `Button size="icon"` default; acceptable for desktop personal tool
- CopyButton icon inside button: `size-3` (12px) — icon only, parent button provides target area

## Typography

All type is Geist Sans (system fallback: `ui-sans-serif, system-ui, sans-serif`). No custom font sizes are declared — shadcn base-nova uses Tailwind's default type scale. The following roles map to that scale:

**Declared weights: 2 — 400 (normal) and 600 (semibold).**

| Role | Tailwind Class | Size | Weight | Line Height | Usage |
|------|---------------|------|--------|-------------|-------|
| Body | `text-sm` | 14px | 400 (normal) | 1.5 | Card field values (`<dd>`, table cells), helper text |
| Label | `text-sm` | 14px | 400 (normal) | 1.5 | Card field labels (`<dt>`), DNS record type badges — differentiated from Body by `<dt>` semantics and position, not weight |
| Heading | `text-base font-semibold` | 16px | 600 (semibold) | 1.2 | Card titles (`CardTitle`), section headings |
| Display | `text-xl font-semibold` | 20px | 600 (semibold) | 1.2 | Page title / app name in header (if present) |

Source: globals.css base layer + shadcn Card component defaults. No overrides needed for this phase.

## Color

All colors are CSS variables resolved by shadcn base-nova neutral preset. Both light and dark values are established in globals.css — Phase 3 adds no new color tokens.

| Role | CSS Variable | Light Value (OKLCH) | Dark Value (OKLCH) | Usage |
|------|-------------|--------------------|--------------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` = white | `oklch(0.145 0 0)` = near-black | Page background, search bar row background |
| Secondary (30%) | `--card` | `oklch(1 0 0)` = white | `oklch(0.205 0 0)` = dark gray | All 4 data cards, popover surfaces |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` = near-black | `oklch(0.922 0 0)` = light gray | Search button fill, focus rings |
| Muted | `--muted-foreground` | `oklch(0.556 0 0)` | `oklch(0.708 0 0)` | Empty state body text, placeholder text |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` = red | `oklch(0.704 0.191 22.216)` = red | Error state card message — no destructive actions in this phase |

Accent reserved for:
- Search / lookup submit button (primary variant)
- Focused input ring
- Active/selected state indicators

The `ghost` variant used for CopyButton and theme toggle uses `--accent` / `--muted` as hover background — NOT the primary accent. This keeps the 10% accent budget intact.

Dark mode: `next-themes` with `attribute="class"` and `defaultTheme="system"` is already wired in `layout.tsx`. Theme toggle switches between `"light"` and `"dark"` via `setTheme`. `disableTransitionOnChange` prevents FOUC. Source: CONTEXT.md D-13 through D-16, RESEARCH.md Pattern 5.

## Interaction Contract

### Copy-to-Clipboard (UI-03)

- Trigger: hover over any value row (`<dd>`, table data cell, PTR value)
- Show: `CopyButton` transitions from `opacity-0` to `opacity-100` via `transition-opacity duration-150`
- Container pattern: `<div className="flex items-center gap-1 group">` wrapping value + `<CopyButton value={...} />`
- Click behavior: `navigator.clipboard.writeText(value)` fires; icon swaps from `Copy` to `Check` (both `size-3`)
- Revert: `setTimeout(() => setCopied(false), 1500)` — 1.5s exactly (CONTEXT.md D-10)
- Labels (`<dt>` text, record type badges): no copy button (CONTEXT.md D-12)
- Button size: `size-6` (24px square) with `variant="ghost"`
- Tailwind hover classes: `opacity-0 group-hover:opacity-100 transition-opacity duration-150`

### Theme Toggle (UI-02)

- Placement: right of `<form>` element, in a `flex items-center gap-2` wrapper row (CONTEXT.md D-13)
- Button: `<Button variant="ghost" size="icon" aria-label="Toggle theme">`
- Icon logic: `resolvedTheme === 'dark'` renders `<Sun className="size-4" />`; otherwise renders `<Moon className="size-4" />`
- Hydration guard: render `null` when `resolvedTheme` is `undefined` (first SSR render) — prevents icon flash (RESEARCH.md Pitfall 2)
- Toggle action: `setTheme(resolvedTheme === 'dark' ? 'light' : 'dark')`
- No background, no border — ghost variant only (CONTEXT.md D-15)

### URL Param — Shareable Links (SRCH-03)

- Write: on form submit, call `router.replace(\`\${pathname}?q=\${encodeURIComponent(input.trim())}\`)` (CONTEXT.md D-01, D-02)
- Read on mount: `const q = searchParams.get('q')` — if present, `setInput(q)` then `lookup(q)` (CONTEXT.md D-03, D-04)
- Suspense requirement: `page.tsx` wraps `<LookupDashboard>` in `<Suspense fallback={null}>` (RESEARCH.md Pattern 1)

### My IP Auto-Load (SRCH-04)

- Trigger: mount with no `?q=` param (CONTEXT.md D-05)
- Fetch: `GET /api/geo` (no q argument); route updated to forward bare ip-api.com call
- Success: `setInput(data.query)` then `lookup(data.query)` — search bar pre-populated and lookup fires (CONTEXT.md D-06)
- Failure: silent — empty search bar, no error shown (CONTEXT.md D-07)
- Effect deps: empty array `[]` — do NOT add `lookup` to deps (RESEARCH.md Pitfall 3)
