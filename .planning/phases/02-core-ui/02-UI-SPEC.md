## Spacing Scale

8-point scale using Tailwind utility classes. All values are multiples of 4.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps (`AlertCircle` to error text), inline badge padding |
| sm | 8px | `gap-2`, `p-2` | Search bar gap between input and button, error text margin-top |
| md | 16px | `gap-4`, `p-4` | Card grid gap, CardContent internal padding |
| lg | 24px | `gap-6`, `p-6` | Card grid top margin from search form |
| xl | 32px | `gap-8`, `py-8` | Page-level top/bottom padding |
| 2xl | 48px | `py-12` | Page top padding (`py-12` in LookupDashboard `<main>`) |
| 3xl | 64px | `py-16` | Reserved â€” not used in Phase 2 |

**Exceptions:**
- Search input field: `px-3 py-2` (12px Ă— 8px) â€” standard text input sizing, not card-level spacing
- Skeleton bars: heights vary per card to approximate real data rows (see Skeleton section below)
- DNS record table rows: `py-1` (4px top+bottom) per row â€” dense tabular data context

## Color

This project uses shadcn's CSS variable system exclusively. No hardcoded hex or oklch values in component code. All references below show the Tailwind utility class that consumes the variable.

| Role | CSS Variable | Tailwind Utility | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `bg-background` | Page background, card background in light mode |
| Secondary (30%) | `--card` | `bg-card` | shadcn Card component surface â€” distinct from page bg in dark mode |
| Accent (10%) | `--primary` | `bg-primary` / `text-primary` | Search button (shadcn Button default variant) only |
| Muted surface | `--muted` | `bg-muted` | Skeleton shimmer bars background |
| Muted text | `--muted-foreground` | `text-muted-foreground` | Field labels, idle state copy, DNS section headers, TTL values |
| Destructive | `--destructive` | `text-destructive` | Validation error text below search bar; card error state icon + text |
| Border | `--border` | `border-border` | Card borders (handled by shadcn Card automatically) |
| Input | `--input` | `border-input` | Search text input border |

**Accent reserved for:** Search button (the single primary action element in Phase 2). No other element uses `bg-primary` or accent-weight color.

**Dark mode:** Handled automatically by `ThemeProvider` (class strategy). The CSS variable values flip between `:root` and `.dark` blocks in `globals.css`. Component code must not write any `dark:` manual overrides â€” the variable system handles it.

## Interaction Contract

| Interaction | Trigger | Behavior | Notes |
|-------------|---------|----------|-------|
| Submit search | Enter key OR click "Search" | `handleSubmit` â†’ `lookup(input)` â†’ parallel API calls | Source: CONTEXT.md |
| Input disabled during load | `isLoading === true` | Input + button both `disabled` | Prevents double-fire |
| Spinner shown | `isLoading === true` | `Loader2` icon with `animate-spin` inside Button | Alongside "Search" text |
| Validation error shown | `detectInputType` returns `'unknown'` OR input is empty | Red text below bar; no API calls | Clears on next submission attempt |
| ReverseDnsCard hidden | Input type is `'domain'` | Removed from DOM (`{inputType !== 'domain' && ...}`) | Grid reflows; not CSS-hidden |
| Card error isolation | One API call fails | Only that card shows error; others unaffected | `Promise.allSettled` guarantee |
